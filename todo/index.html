    Document Bayang TX

    The protocol sends out a TX packet every 5 milliseconds. Every millisecond after sending the TX packet it checks to see if the flight controller has sent back a telemetry packet. The FC only sends back one telemetry packet for each TX packet it receives.

    We configure the microcontroller to trigger an interrupt subroutine (ISR) every millisecond. The send TX, receive telemetry actions are performed on one of these interrupts. Because this work is performed by an interrupt subroutine we must take care:

    • Any hardware we interface with must not be in use by the main thread to avoid any contentions/conflicts between the ISR and the main thread.
    • The ISR must be careful not to call any code that isn't thread safe. The micros_realtime() function is an example of code that isn't thread safe. That is why we implement a separate micros() function for exclusive use by the ISR code.
